# full-stack roadmaps

## Roadmap Développeur Full-Stack Senior

1. Architecture logicielle   - Clean Architecture   - Hexagonal Architecture   - Onion Architecture   - DDD (Domain Driven Design) 
    - CQRS   - Event Sourcing   - Repository Pattern   - Unit of Work   - Dependency Injection
2. Design Patterns   - Factory   - Abstract Factory   - Builder   - Strategy   - Observer   - Command   - Decorator  
   - Adapter   - Facade   - Singleton (quand l'utiliser)   - Proxy   - Chain of Responsibility
3. Principes de conception   - SOLID   - DRY   - KISS   - YAGNI   - GRASP   - Tell Don't Ask   - Law of Demeter
4. API   - REST   - GraphQL   - gRPC   - OpenAPI / Swagger   - Versionning   - Pagination   - Rate Limiting   - Idempotence
5. Authentification   - JWT   - OAuth2   - OpenID Connect   - Sessions   - Refresh Token   - MFA   - SSO
6. Sécurité   - OWASP Top 10   - XSS   - CSRF   - SQL Injection   - SSRF   - CSP   - CORS   - Hashing   - Chiffrement
7. Docker   - Dockerfile   - Multi-stage Build   - Docker Compose   - Volumes   - Networks   - Healthchecks
8. Kubernetes   - Pods   - Deployments   - Services   - ConfigMaps   - Secrets   - Ingress   - Autoscaling
9. CI/CD   - GitHub Actions   - GitLab CI   - Jenkins   - Pipelines   - Tests automatiques   - Déploiement continu
10. Monitoring    - Prometheus    - Grafana    - Loki    - ELK Stack    - OpenTelemetry    - Sentry
11. Cloud    - AWS    - Azure    - Google Cloud    - IAM    - S3    - CloudFront    - Load Balancer
12. Performance    - Caching    - CDN    - Compression    - Profiling    - Benchmark    - Optimisation SQL    - Optimisation Front
13. Messagerie    - RabbitMQ    - Kafka    - Redis Streams    - Dead Letter Queue    - Retry
14. Architecture distribuée    - Microservices    - Monolithe modulaire    - Saga Pattern    - API Gateway    - Service Discovery
15. Tests    - Unitaires    - Intégration    - Fonctionnels    - End-to-End    - Tests de charge    - Mutation Testing
16. Git avancé    - Rebase    - Cherry-pick    - Reflog    - Bisect    - Hooks    - Git Flow    - Trunk Based Development
17. Base de données    - PostgreSQL    - MySQL    - Redis    - MongoDB    - Indexation    - Transactions    - Optimisation
18. Communication    - WebSockets    - SSE    - HTTP/2    - HTTP/3    - gRPC
19. Observabilité    - Logs    - Metrics    - Traces    - Correlation ID
20. Soft Skills    - Code Review    - Mentorat    - Documentation    - Estimation    - Architecture Decision Records (ADR)    - Communication


## must have

1. Docker (très bonne maîtrise).
2. GitHub Actions ou un autre outil de CI/CD.
3. Architecture (DDD, Clean Architecture, CQRS).
4. RabbitMQ ou Kafka pour la messagerie.
5. Kubernetes (au moins les bases).
6. Observabilité (Prometheus, Grafana, Sentry, OpenTelemetry).
7. AWS (ou un autre cloud majeur).
8. Performance et optimisation (profiling, cache, index, HTTP).

## bonus

- Lire les RFC HTTP et comprendre le protocole (méthodes, statuts, cache, ETag, compression, négociation de contenu).
- Comprendre TCP/IP, DNS, TLS et le fonctionnement d'une requête du navigateur jusqu'au serveur.
- Connaître le fonctionnement de la mémoire, du multithreading, des processus et de la concurrence.
- Savoir profiler une application pour identifier un goulot d'étranglement (CPU, mémoire, I/O, réseau).
- Concevoir une architecture adaptée au contexte, en sachant quand rester sur un monolithe plutôt que de passer aux microservices.
- Écrire du code maintenable, avec des tests pertinents et une documentation utile.
